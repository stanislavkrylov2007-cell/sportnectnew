Функциональные требования

FR-1​

Регистрация/вход на сайт

FR-2​

После входа на аккаунт появляется меню:

  1. Найти свободные игры на карте
  2. Найти свободные корты на карте, чтобы создать событие
  3. Уведомления
  4. Поддержка

FR-3​

Запись на событие и подтверждение (если игрок)

FR-4​

Создание события (если организатор)

FR-5​

После завершения события участники могут оценить прошедшую игру по шкале 1–5 и при желании оценить отдельных игроков, что влияет на их рейтинг.

FR-6​

В разделе «Уведомления» будут отображаться список уведомлений по типу ("Заявка на игру принята")

FR-7​

В разделе «Поддержка» отображается форма обратной связи:

  1. Тема
  2. Текст сообщения
  3. Кнопка «Отправить»

Use Case (для 4 основных функций)​

UC-1: Регистрация/вход​

ПолеЗначение
IDUC-1
НазваниеРегистрация и вход в систему
АкторыНовый или существующий пользователь
ПредусловияПользователь открыл сайт, не авторизован

Основной поток:

  1. Выбор действия — Пользователь выбирает «Вход» или «Регистрация»

  2. Вход (если выбран вход)
    2.1 Система показывает форму входа (почта, пароль)
    2.2 Пользователь вводит почту и пароль
    2.3 Система проверяет данные
    2.4 Если данные верны — пользователь авторизован и перенаправлен в главное меню

  3. Регистрация (если выбрана регистрация)
    3.1 Система показывает форму регистрации (почта, пароль, уровень игры)
    3.2 Пользователь вводит данные и выбирает уровень
    3.3 Система проверяет, что почта не занята
    3.4 Система создаёт учётную запись
    3.5 Система автоматически авторизует пользователя
    3.6 Пользователь перенаправлен в главное меню

Постусловия: Пользователь успешно авторизован в системе.

Альтернативный поток:

  • 2.3а Неверный пароль → Система сообщает «Неверный пароль» и предлагает повторить или восстановить пароль
  • 3.3а Почта уже занята → Система сообщает «Эта почта уже зарегистрирована» и предлагает войти

Исключительный поток:

  • 3.4a Технический сбой → Система сообщает «Не удалось создать аккаунт. Попробуйте позже» и логирует ошибку для разработчика.

UC-2: Создание события​

ПолеЗначение
IDUC-2
НазваниеСоздание события
АкторыАвторизованный пользователь (организатор)
ПредусловияПользователь авторизован, открыл раздел «Найти свободные корты»

Основной поток:

  1. Пользователь выбирает площадку на карте
  2. Система показывает информацию о площадке и кнопку «Создать событие здесь»
  3. Пользователь нажимает кнопку
  4. Система открывает форму создания события:
    • Уровень игры (выбор)
    • Дата и время
    • Количество участников
    • Цена с человека (если корт платный)
  5. Пользователь заполняет поля и нажимает «Опубликовать»
  6. Система проверяет заполнение обязательных полей
  7. Система сохраняет событие
  8. Событие появляется на карте в разделе «Найти свободные игры»

Постусловия: Событие создано, доступно для просмотра и записи

Альтернативный поток:

  • 6а Обязательные поля не заполнены → Система подсвечивает незаполненные поля и не даёт опубликовать

Исключительный поток:

  • 3.4a Технический сбой → Система сообщает «Не удалось создать событие. Попробуйте позже» и логирует ошибку

UC-3: Запись на событие и подтверждение​

ПолеЗначение
IDUC-3
НазваниеЗапись на игру и подтверждение организатором
АкторыИгрок, Организатор
ПредусловияОба пользователя авторизованы, событие существует и активно

Основной поток:

1. Запись игрока
1.1 Игрок находит событие на карте и открывает карточку
1.2 Игрок нажимает «Записаться»
1.3 Система проверяет наличие свободных мест
1.4 Система добавляет игрока в список участников со статусом «ожидает подтверждения»
1.5 Система отправляет уведомление организатору
1.6 Игрок видит сообщение «Заявка отправлена»

2. Подтверждение организатором
2.1 Организатор открывает карточку события
2.2 Организатор видит список заявок
2.3 Организатор нажимает «Подтвердить» напротив заявки игрока
2.4 Система меняет статус игрока на «подтверждено»
2.5 Система отправляет уведомление игроку «Заявка подтверждена»

Постусловия: Игрок подтверждён, место занято

Альтернативный поток:

  • 1.3а Мест нет → Система сообщает «Свободных мест нет» и предлагает добавиться в лист ожидания
  • 2.3а Организатор отклоняет заявку → Статус меняется на «отклонено», уведомление игроку, место освобождается

UC-4: Оценка прошедшего события​

ПолеЗначение
IDUC-4
НазваниеОценка прошедшего события и игроков
АкторыУчастник события (игрок, организатор)
ПредусловияСобытие завершено (дата и время прошли), пользователь был участником

Основной поток:

  1. Система отправляет уведомление с предложением оценить событие
  2. Пользователь открывает карточку завершённого события
  3. Система отображает форму оценки
  4. Пользователь ставит общую оценку событию (1–5) — обязательно
  5. (Опционально) Пользователь может оценить каждого участника (1–5)
  6. (Опционально) Пользователь может оставить текстовый отзыв
  7. Пользователь нажимает «Отправить»
  8. Система сохраняет оценки и обновляет рейтинг события и игроков

Постусловия: Оценка сохранена, рейтинг обновлён

Альтернативный поток:

  • 4а Пользователь пропускает оценку → система напоминает через день, но не более 3 раз
